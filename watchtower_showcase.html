// Architecture

How It Works

Six components working in harmony — fully autonomous, no manual trigger needed.

01 / TRIGGER
⏰

Auto-Schedule

APScheduler fires scan_emails() every 2 hours automatically. Runs 24/7 on Railway cloud even when your PC is off.

APScheduler · Python
02 / FETCH
📧

Gmail Reader

Gmail API with OAuth2 fetches last 10 unread emails. Extracts subject, sender, and body from nested MIME structure.

Gmail API · OAuth2
03 / CLASSIFY
🧠

AI Classifier

Each email sent to Llama 3.1 8B via Groq API. Returns CATEGORY (URGENT/ROUTINE/IGNORE), REASON, and ACTION.

Groq API · LangChain · Llama 3.1 8B
04 / REPLY
✍️

Auto-Reply

If recruiter asks for resume/skills, LLM generates professional reply using profile data. Sends email with resume PDF attached.

LangChain · Gmail API · MIME
05 / NOTIFY
🔔

Discord Alert

Urgent emails trigger instant Discord DM with full analysis. Free forever — no expiry unlike WhatsApp sandbox.

Discord Bot API · discord.py
06 / REPORT
📊

Scan Summary

After every scan, sends Discord summary: urgent count, auto-replied count, routine count, total scanned, next scan time.

Discord Bot API

ChatGroq integration for structured LLM calls with consistent output formatting.

Cloud PaaS deployment. Auto-deploys from GitHub. Runs as background worker 24/7.

Core language. APScheduler for timing, asyncio for Discord, dotenv for secrets.
